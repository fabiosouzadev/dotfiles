## Task Boundary

Ajustar o script de AI tools (`home/.chezmoiscripts/unix/run_once_after_510-install-ai-tools.sh.tmpl`)
para que a instalação do "ibm bob" (entrada `bobibm` em `home/.chezmoidata/ai_tools.yaml`, curl tool)
ocorra apenas em ambientes diferentes de Zup. Na Zup a instalação é bloqueada.

Ambiente Zup é detectado por `.settings.workplace == "Zup"`
(definido em `home/.chezmoi.yaml.tmpl:241` via `ternary "Zup" "none" (eq $profile "work")`).

</domain>

<decisions>
## Implementation Decisions

### Mecanismo do guard
- Adicionar um campo declarativo na entrada da ferramenta no `ai_tools.yaml`
  (ex.: `block_on_workplace: Zup`).
- O script de instalação respeita esse campo genericamente para qualquer tool
  (npm e curl), pulando a instalação quando `$.settings.workplace` for igual ao valor do campo.
- Não usar guard hardcoded para `bobibm` — a solução deve ser reutilizável.

### Escopo de ambientes
- `bobibm` instala em **todos os workplaces exceto Zup** (none/pessoal, work não-Zup).
- Bloqueia exclusivamente quando `workplace == "Zup"`.

### Ambientes efêmeros
- JÁ COBERTOS. O script começa com `{{ template "common/script_is_not_ephemeral" . }}`
  (linha 3), que dá `exit 0` quando `.settings.ephemeral` é verdadeiro.
- Nenhuma alteração necessária para efêmeros — todas as tools já são puladas nesse caso.

### Claude's Discretion
- Nome exato do campo (`block_on_workplace`) e sintaxe do guard no template Go.
- Posição do campo dentro da entrada `bobibm`.
- Se aplicar a checagem em ambos os ranges (npm + curl) ou apenas no curl —
  recomendação: aplicar em ambos para consistência, já que o campo é genérico.

</decisions>
